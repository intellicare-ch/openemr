# Data Exchange and Portability

**Capability ID:** 23
**Last Updated:** 2025-11-15

## Overview
Patient data export and import capabilities supporting bulk FHIR export, EHI exporter for patient requests, CCDA import/export, data de-identification, and external data integration.

## Key Features
- Bulk FHIR export ($export operation)
- EHI (Electronic Health Information) exporter
- Patient-requested data export
- CCDA import and export
- Data de-identification tool
- Re-identification capability
- External data import
- CSV/Excel data import
- Database export
- Backup and restore
- Data migration tools
- API-based data exchange

**Files:** `/interface/modules/custom_modules/oe-module-ehi-exporter/`, `/interface/de_identification_forms/`, `/ccdaservice/`
**Module:** EHI Exporter module
**Standards:** FHIR Bulk Data IG, CCDA 2.1, 21st Century Cures Act

## Configuration
- Export formats available
- De-identification rules
- Patient export permissions
- Bulk export settings

## Compliance
- **21st Century Cures Act** - Patient data access and portability
- **ONC** - EHI export requirements
- **HIPAA** - De-identification standards (Safe Harbor method)
